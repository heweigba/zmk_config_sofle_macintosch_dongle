## Position码计算方法

ZMK的position码是按照映射表的顺序递增的：
- **Position 0** = 映射表的第1个位置
- **Position 1** = 映射表的第2个位置
- **Position N** = 映射表的第N+1个位置

## 问题分析

### 为什么测试结果是"长按H键切换鼠标层"？

**代码中的监听**：
```c
// trackpoint_0x15.c:65
if (ev->position == 35) { // J key position
```

**理论上**：
- 接收器版中 position 35 = J键

**但用户测试**：
- 长按H键触发切换 → 说明position 35实际对应H键

**矛盾！** 两个结论冲突了。

---

## 可能的原因

### 原因1：第0行有额外的键

## 修正后的结论

### 接收器版的正确position码：

| 按键 | Position码 | 原代码使用的Position | 状态 |
|------|-----------|---------------------|------|
| **H键** | **35** | - | ✅ 与测试结果一致 |
| **J键** | **36** | **35** | ❌ **错误！偏移了1位** |
| **K键** | **37** | - | ✅ |
| **L键** | **38** | - | ✅ |

---

## 根本原因

**接收器版的第0行多了一个bootloader键（RC(0,0)在右上角），导致整个position映射表向下偏移了1位！**

- 普通版：J键 = position 35
- 接收器版：J键 = position **36**（因为第0行多了1个键）
- 代码中监听 position 35，实际监听到的是 **H键**

---

## 修复方案

### 文件：`trackpoint_0x15.c`

**第65行修改**：
```c
// ❌ 错误（普通版的position）
if (ev->position == 35) { // J key position

// ✅ 正确（接收器版的position）
if (ev->position == 36) { // J key position (接收器版偏移+1)
```
