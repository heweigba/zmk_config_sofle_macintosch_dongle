# Split连接问题排查指南

## 问题现象
- 先刷接收器 → 小红点能用
- 再刷左右固件 → 都不能用了
- 没有识别到设备

## 根本原因
**左右键盘与接收器的split连接没有建立**

可能原因：
1. 未刷settings_reset，旧配置干扰
2. 蓝牙配对信息丢失
3. 固件不匹配（左右手固件与接收器固件版本不一致）

## 正确的完全重置刷写流程

### 准备工作
下载最新firmware.zip，解压后确认有以下文件：
```
firmware/
├── sofle_dongle/
│   └── sofle_dongle-u2f.bin              # 接收器
├── sofle_dongle_left/
│   ├── left_bbtrackball_dongle-u2f.bin   # 左手键盘
│   └── settings_reset-u2f.bin            # 左手重置
├── sofle_dongle_right/
│   ├── right_trackpoint_dongle-u2f.bin   # 右手键盘
│   └── settings_reset-u2f.bin            # 右手重置
```

### 第1步：完全重置右手键盘
```
1. 右手键盘进入bootloader模式
2. 刷入：settings_reset-u2f.bin（清除所有旧配置）
3. 重启右手键盘
4. 再次进入bootloader模式
5. 刷入：right_trackpoint_dongle-u2f.bin
6. 重启右手键盘
```

### 第2步：完全重置左手键盘
```
1. 左手键盘进入bootloader模式
2. 刷入：settings_reset-u2f.bin（清除所有旧配置）
3. 重启左手键盘
4. 再次进入bootloader模式
5. 刷入：left_bbtrackball_dongle-u2f.bin
6. 重启左手键盘
```

### 第3步：重置接收器（可选）
如果接收器已经工作正常，可以跳过。
```
1. 接收器进入bootloader模式
2. 刷入：sofle_dongle-u2f.bin
3. 重启接收器
```

### 第4步：验证连接
```
1. 接收器连接电脑（USB或蓝牙）
2. 等待10-20秒，让左右手自动连接
3. 打开文本编辑器
4. 按左右手键盘的按键，看是否输入字符
5. 测试轨迹球和小红点
```

## 验证Split连接是否成功

### 方法1：观察LED
- 左右手键盘LED应该常亮或规律闪烁（表示已连接）
- 如果LED快速闪烁或熄灭，表示未连接

### 方法2：测试按键
- 按左手键盘的字母键，看是否输入字符
- 按右手键盘的字母键，看是否输入字符
- 如果按键有反应，说明split连接已建立

### 方法3：查看接收器输出
- 移动小红点，看光标是否移动
- 滚动轨迹球，看是否发送方向键

## 如果还是不行

### 检查1：固件版本是否一致
- 确保所有固件来自同一个GitHub Actions构建
- 不要混用不同构建的固件

### 检查2：接收器是否进入配对模式
- 某些情况下，接收器需要手动进入配对模式
- 查看接收器LED状态，确认配对状态

### 检查3：单独测试
```
1. 只连接接收器，不连接左右手
2. 用USB线直接连接右手键盘到电脑
3. 测试小红点是否能用（验证右手固件本身正常）
4. 同样测试左手键盘
```

### 检查4：查看ZMK日志（需要调试工具）
如果有USB串口调试工具，可以查看日志输出。

## 常见问题

### Q: 为什么一定要先刷settings_reset？
A: 新固件可能有不同的配置结构或蓝牙配对信息，旧配置会干扰新固件正常工作。

### Q: 刷写顺序重要吗？
A: 重要！必须先刷外设（左右手），最后刷中心（接收器），或者只刷外设。

### Q: 可以只刷一边吗？
A: 可以。如果只有一边有问题，只重置并刷写那一边即可。

## 关于自动层切换

split连接建立后，自动层切换应该自动工作：
- 移动小红点 → 自动进入鼠标层（层2）
- 停止400ms → 自动切回默认层（层0）

如果split连接没建立，自动层切换自然不会触发。
