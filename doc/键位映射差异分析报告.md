# 接收器版键位映射差异分析报告

## 对比范围
- **普通版**：zmk_config_zitaotech_sofle
- **接收器版**：zmk_config_sofle_macintosch_dongle
- **对比层级**：第0层（default_layer）、第2层（mouse_layer）

## 第0层（default_layer）差异分析

### 差异对照表

| 位置 | 普通版 | 接收器版 | 状态 |
|------|--------|---------|------|
| **右上角** | 无键 | **&BOT**（bootloader） | ⚠️ 接收器版特有 |
| 第0行中间左 | &kp C_MUTE | &kp C_MUTE | ✅ 一致 |
| 第0行中间右 | **&kp LC(LS(K_PWR))** | **&kp C_MUTE** | ❌ **不同** |
| 第2行最左 | **&lt 1 CAPS**（长按切换func层） | **&kp CAPS**（普通CAPS） | ❌ **不同** |
| 第2行中间左 | &kp ENTER | **&mkp LCLK**（鼠标左键） | ❌ **不同** |
| 第2行中间右 | &kp F14 | **&mkp LCLK**（鼠标左键） | ❌ **不同** |
| 第3行中间左 | &kp PG_DN | &kp PG_DN | ✅ 一致 |
| 第3行中间右 | &kp PG_UP | &kp PG_UP | ✅ 一致 |
| 第3行右手第1键 | &kp F13 | **&mkp LCLK**（鼠标左键） | ❌ **不同** |
| 第3行右手第2键 | &kp F14 | **&mkp RCLK**（鼠标右键） | ❌ **不同** |
| 第4行左手第3键 | &kp LGUI | &kp LGUI | ✅ 一致 |
| 第4行左手第4键 | **&mo 1**（切换func层） | &kp SPACE | ❌ **不同** |

| 第4行右手第1键 | &mo 1 | &mo 1 | ✅ 一致 |
| 第4行右手第2键 | &kp SPACE | &kp SPACE | ✅ 一致 |
| 第4行右手第3键 | **&kp RIGHT_SHIFT** | **&kp RGUI** | ❌ **不同** |
| 第4行右手第4键 | &kp RALT | &kp RALT | ✅ 一致 |
| 第4行右手第5键 | &kp RCTRL | &kp RCTRL | ✅ 一致 |

### 关键差异说明

#### 1. 第0行中间右键：K_PWR vs C_MUTE
- **普通版**：`&kp LC(LS(K_PWR))`（Cmd+Shift+Power = macOS锁屏）
- **接收器版**：`&kp C_MUTE`（静音）
- **影响**：接收器版缺少macOS锁屏快捷键

#### 2. 第2行CAPS键：长按切换层 vs 普通CAPS
- **普通版**：`&lt 1 CAPS`（长按切换到func层，短按CAPS）
- **接收器版**：`&kp CAPS`（普通CAPS）
- **影响**：接收器版无法通过长按CAPS快速进入func层

#### 3. 第2行中间键：ENTER/F14 vs 鼠标左键
- **普通版**：中间左=`&kp ENTER`，中间右=`&kp F14`
- **接收器版**：中间左=`&mkp LCLK`，中间右=`&mkp LCLK`（都是鼠标左键）
- **影响**：接收器版在default层就可以使用鼠标左键

#### 4. 第3行右手前两键：F13/F14 vs 鼠标左右键
- **普通版**：右手第1=`&kp F13`，右手第2=`&kp F14`
- **接收器版**：右手第1=`&mkp LCLK`，右手第2=`&mkp RCLK`
- **影响**：接收器版在default层有完整的鼠标左右键功能

#### 5. 第4行左手Space和mo 1位置互换
- **普通版**：左手第4=`&mo 1`，左手第5=`&kp SPACE`
- **接收器版**：左手第4=`&kp SPACE`，左手第5=`&mo 1`
- **影响**：func层切换键位置不同

#### 6. 第4行右手第3键：RIGHT_SHIFT vs RGUI
- **普通版**：`&kp RIGHT_SHIFT`（右Shift）
- **接收器版**：`&kp RGUI`（右Cmd）
- **影响**：接收器版没有右Shift，改为右Cmd

### 关键差异说明

#### 1. 右上角bootloader键
- **接收器版**：多了一个`&BOT`键在右上角
- **影响**：方便进入bootloader模式刷固件

#### 2. sensor绑定不同
- **普通版**：右encoder使用`&scroll_accel`（自定义滚轮加速器）
- **接收器版**：右encoder使用`&inc_dec_kp LEFT RIGHT`（左右方向键）
- **影响**：接收器版的右encoder在mouse层是方向键，不是滚轮

#### 3. 其他键位
- **状态**：除sensor外，所有键位完全一致 ✅

## 问题总结

### 第0层主要问题

1. ❌ **缺少macOS锁屏快捷键**：K_PWR → C_MUTE
2. ❌ **CAPS无法长按切换func层**：`&lt 1 CAPS` → `&kp CAPS`
3. ⚠️ **中间键功能不同**：ENTER/F14 → 鼠标左键（可能是设计差异）
4. ⚠️ **Space和mo 1位置互换**（可能是设计差异）
5. ⚠️ **缺少右Shift**：RIGHT_SHIFT → RGUI（可能是设计差异）

### 第2层主要问题

1. ❌ **sensor绑定不同**：`&scroll_accel` → `&inc_dec_kp LEFT RIGHT`
2. ⚠️ **右上角多了一个bootloader键**（设计差异，不影响使用）

---

## 建议修复方案

### 方案A：完全对齐普通版

**第0层修复**：
1. 第0行中间右：`&kp C_MUTE` → `&kp LC(LS(K_PWR))`（macOS锁屏）
2. 第2行最左：`&kp CAPS` → `&lt 1 CAPS`（长按切换func层）
3. 第2行中间：`&mkp LCLK` → `&kp ENTER` / `&kp F14`
4. 第3行右手前两键：`&mkp LCLK` / `&mkp RCLK` → `&kp F13` / `&kp F14`
5. 第4行左手：`&kp SPACE` / `&mo 1` → `&mo 1` / `&kp SPACE`（位置互换）
6. 第4行右手第3键：`&kp RGUI` → `&kp RIGHT_SHIFT`

**第2层修复**：
1. sensor绑定：`&inc_dec_kp LEFT RIGHT` → `&scroll_accel`

### 方案B：保留接收器版特色功能

**保留差异**（接收器版优势）：
- ✅ 第0层中间有鼠标左键（方便使用）
- ✅ 第0层右手有鼠标左右键（方便使用）
- ✅ 右上角有bootloader键（方便刷固件）

**仅修复关键功能**：
1. ✅ 第0行中间右：添加macOS锁屏快捷键
2. ✅ 第2行最左：改为`&lt 1 CAPS`（长按切换func层）
3. ✅ 第2层sensor：改为`&scroll_accel`（如果需要滚轮加速）

## 需要确认的问题

1. **第0层的鼠标键是设计差异还是错误？**
   - 如果是设计差异，接收器版在default层就可以用鼠标
   - 如果是错误，应该改为F13/F14等功能键

2. **右Shift改为右Cmd是设计差异还是错误？**
   - macOS用户可能更喜欢右Cmd
   - 但失去了右Shift功能

3. **Space和mo 1位置互换是设计差异还是错误？**
   - 可能是为了更方便切换func层

4. **第2层sensor绑定不同是有意为之吗？**
   - 普通版用滚轮加速器
   - 接收器版用方向键
